¿Por qué las imágenes, firmas, audios y videos no persisten al cambiar de computador?

El navegador tiene políticas de seguridad estrictas (Sandbox) al abrir archivos HTML directamente desde el disco (protocolo file://).

  • Límite de LocalStorage (5MB): El almacenamiento local estándar es muy limitado. Guardar audios, fotos y videos de alta resolución en él haría que el navegador colapse por exceso de cuota.
  • IndexedDB local: Para evitar colapsos, la herramienta guarda temporalmente las fotos, audios y videos en una base de datos local del navegador llamada IndexedDB. Sin embargo, esta base de datos reside únicamente en el computador y navegador donde cargaste el archivo. Si abres el archivo HTML en otro computador, otra sesión o modo Incógnito, el navegador no encontrará ese almacenamiento local.
  • Firmas digitalizadas: Las firmas se guardan directamente como datos vectoriales Base64 dentro del archivo HTML al exportar, pero para garantizar su correcta inicialización y evitar pérdidas de datos en diferentes dispositivos, siempre te recomendamos presionar "Exportar HTML con Cambios" antes de cerrar la pestaña.
Recomendación Profesional SG-SST:

Para asegurar una gestión documental robusta y duradera de este Reglamento Interno de Trabajo (RIT), te recomendamos crear una carpeta en tu computador con el nombre del caso (Ej: Reglamento_Interno_Trabajo) y guardar allí:

  • El archivo HTML exportado (que contiene los textos, plan Kanban y firmas digitalizadas).
  • Los archivos originales de las fotos o soportes físicos anexados.
  • Los archivos de audio explicativos de las jornadas o políticas.
  • Los videos de soporte de socialización si aplica.
