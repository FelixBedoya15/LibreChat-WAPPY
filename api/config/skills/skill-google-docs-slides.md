# Generación de Documentos e Informes (Google Docs & Slides Engine)

Cuando el usuario solicite estructurar o generar un informe técnico editable (Google Docs) o una presentación de resultados corporativa (Google Slides) sobre SST, investigaciones de accidentes, auditorías o avances del sistema de gestión, sigue estas directrices:

## 📄 ESTRUCTURA PARA GOOGLE DOCS (Informes y Procedimientos)

1. **Encabezado Institucional:**
   - Título del Documento, Nombre de la Empresa, Fecha de Elaboración, Versión del Documento y Elaborado por (Agente / Usuario).
2. **Estructura Jerárquica:**
   - Usar estrictamente Título 1 (`#`), Título 2 (`##`) y Título 3 (`###`) para mantener el árbol de navegación navegable en Google Docs.
3. **Secciones Estándar:**
   - `1. Objetivo y Alcance`
   - `2. Marco Normativo Aplicable`
   - `3. Diagnóstico / Hallazgos`
   - `4. Matriz / Plan de Acción`
   - `5. Conclusiones y Firmas de Responsables`

---

## 🖥️ ESTRUCTURA PARA GOOGLE SLIDES (Presentaciones Ejecutivas)

1. **Diapositiva 1 (Portada):** Título de la presentación, Subtítulo y Nombre del Expositor/Área.
2. **Diapositiva 2 (Agenda):** 3 a 5 puntos clave a tratar.
3. **Diapositivas de Contenido:**
   - **Regla del 6x6:** Máximo 6 viñetas por diapositiva y máximo 6 palabras por viñeta. Evitar párrafos largos de texto.
   - Usar resaltados en negrita para métricas clave (ej. **94% Cumplimiento**, **0 Accidentes Mortales**).
4. **Diapositiva Final (Plan de Acción / Compromisos):** Tabla o checklist con responsables y fechas límite de cumplimiento.
